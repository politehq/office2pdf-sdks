# Contributing to Office2PDF SDKs

Thank you for your interest in contributing to Office2PDF.  
We welcome contributions that improve quality, stability, and developer experience.

---

## Scope of Contributions

This repository contains the **official SDKs** for the Office2PDF API.

You are welcome to contribute:

- Bug fixes
- Documentation improvements
- Tests
- New SDKs (Python, Go, Java/Kotlin) in coordination with the maintainers

Please open an issue before starting large changes or new SDK implementations.

---

## Development Guidelines

- Follow existing API conventions and error models
- Keep SDK behavior consistent across languages
- Prefer clarity and correctness over cleverness
- Add tests for new functionality when applicable

---

## Commit Messages

Please use clear, conventional commit messages. Examples:

- `feat(node-sdk): add streaming download support`
- `fix(python-sdk): handle timeout correctly`
- `docs: update README`
- `chore(release): bump version to v0.1.2`

---

## Pull Request Process

1. Fork the repository
2. Create a feature branch from `main`
3. Make your changes with clear commits
4. Open a Pull Request describing:
   - What changed
   - Why it is needed
   - Any breaking behavior (if applicable)

We may request changes before merging to ensure consistency across SDKs.

---

## Questions & Discussions

For general questions, usage help, or SDK requests, please open a GitHub issue.

Thank you for contributing to Office2PDF ❤️
